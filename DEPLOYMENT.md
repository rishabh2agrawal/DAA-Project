# ClimateIQ Deployment Guide

This guide explains how to run and deploy the project-ready v1 of ClimateIQ Bengaluru.

## Local Setup

From the project root:

```bash
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

Build the processed dataset and train models:

```bash
.\.venv\Scripts\python.exe data_processing.py
.\.venv\Scripts\python.exe ai_models\train_extended.py
```

Start backend:

```bash
.\.venv\Scripts\python.exe -m uvicorn backend.app:app --host 127.0.0.1 --port 8000
```

Start frontend in a second terminal:

```bash
.\.venv\Scripts\streamlit.exe run frontend/app.py --server.address 127.0.0.1 --server.port 8501
```

Open:

- Dashboard: `http://127.0.0.1:8501`
- API docs: `http://127.0.0.1:8000/docs`

## Render Deployment

Deploy the backend and frontend as two separate Python web services.

### Backend Service

Build command:

```bash
pip install -r requirements.txt && python data_processing.py && python ai_models/train_extended.py
```

Start command:

```bash
uvicorn backend.app:app --host 0.0.0.0 --port $PORT
```

### Frontend Service

Build command:

```bash
pip install -r requirements.txt
```

Start command:

```bash
streamlit run frontend/app.py --server.address 0.0.0.0 --server.port $PORT
```

Environment variable:

```text
BACKEND_URL=https://your-backend-service.onrender.com
```

## Streamlit Community Cloud

Use `frontend/app.py` as the main file. The FastAPI backend must be deployed separately, then configured in Streamlit secrets or environment variables as `BACKEND_URL`.

## Deployment Notes

- Model files are generated by `ai_models/train_extended.py`.
- `MODEL_RESULTS.md` and `ai_models/saved_models/model_metrics.json` document evaluation output.
- The app is presentation-ready but not publication-grade without official IMD, CPCB, ERA5, Bhuvan/NRSC, Sentinel/Landsat, and health datasets.
